# Frontend Design

Create distinctive, production-grade frontend interfaces. Implement **working code** with clear aesthetic direction—avoid generic "AI slop" (Inter, purple-on-white, predictable layouts).

## Before coding: design direction

1. **Purpose & audience**: What does the interface solve? Who uses it?
2. **Tone**: Pick one and commit—e.g. brutalist minimal, maximalist, retro-futuristic, luxury/refined, editorial, soft/pastel, industrial. Intentionality over intensity.
3. **Differentiation**: One thing that makes it memorable.
4. **Constraints**: Framework, performance, accessibility.

Then implement code that is production-grade, visually striking, cohesive, and refined in detail.

## Aesthetics guidelines

- **Typography**: Distinctive, characterful fonts. Pair a display font with a refined body font. Avoid Inter, Roboto, Arial, system fonts, Space Grotesk.
- **Color**: Cohesive palette via CSS variables. Dominant colors + sharp accents; avoid timid or clichéd schemes (e.g. purple gradients on white).
- **Motion**: Animations for impact—staggered reveals, scroll/hover. Prefer CSS for HTML; Motion library for React when available. One strong moment beats many weak ones.
- **Layout**: Asymmetry, overlap, diagonal flow, grid-breaking elements. Generous negative space or controlled density.
- **Backgrounds & depth**: Atmosphere over flat solids—gradient meshes, noise, patterns, layered transparency, shadows, grain. Match the chosen aesthetic.

**Avoid**: Overused fonts, clichéd color schemes, cookie-cutter layouts, same look across generations. Vary themes (light/dark), fonts, and aesthetics per request.

**Match complexity to vision**: Maximalist → richer code and effects. Minimalist → restraint, precision, spacing and typography focus.

## Output

Deliver real, runnable code (HTML/CSS/JS, React, Vue, etc.) that reflects the chosen direction. No two designs should look the same; commit fully to a single, distinctive vision per task.
