# 자산 한눈에 관리 및 월별 검증 기능 PRD

여러 통장/계좌를 등록해 총 자산을 관리하고, "3월부터" 기준일 잔액을 두어 월별 지출·수입과 현재 보유 자산이 일치하는지 검증할 수 있는 기능에 대한 제품 요구사항 정리입니다.

---

## 1. 가계부 계산 논리 (검증용 수식)

거래가 모두 **한 계정(accountId)** 에만 묶여 있다고 할 때, 한 계정에 대해:

```
계산 잔액 = 기준일 잔액
         + (기준일 이후 ~ 오늘) 해당 계정 수입 합계
         - (기준일 이후 ~ 오늘) 해당 계정 지출 합계
         + (해당 계정으로 들어온 이체) - (해당 계정에서 나간 이체)
```

- **이체 미구현 시**: 위 식에서 이체 항은 0. 수입/지출만 반영.
- **총 자산 검증**
  - `총 자산(입력) = Σ 각 계정 입력 잔액`
  - `총 자산(계산) = Σ 각 계정 계산 잔액`
  - 두 값이 같으면 "딱 떨어짐". 다르면 어느 계정에서 차이가 나는지 표시.

**월별로 끊어 보는 경우 (예: 3월)**

- 3월 1일 0시 **기준일 잔액**: 사용자가 계정별로 한 번 입력.
- 3월 중 **수입/지출**: 이미 거래로 입력됨.
- 3월 말 **계산 잔액** = 3월 1일 기준일 잔액 + 3월 수입 - 3월 지출 (이체 제외 시).
- "3월 말 입력 잔액"을 사용자가 입력해 둔다면, **입력 잔액 vs 3월 말 계산 잔액** 비교로 검증 가능.

정리하면, **기준일 잔액**을 하나 정해 두고, **계정별·기간별(또는 "현재") 계산 잔액**을 위 식으로 구한 뒤, **입력 잔액과 비교**하는 흐름이 필요합니다.

## 2. 구현 요약

### 2-1. 데이터 모델

- **Account** (Prisma)
  - `initialBalanceDate: DateTime?` — 기준일 (예: 2025-03-01)
  - `initialBalance: Decimal?` — 그날 0시 잔액

### 2-2. API

- **계정 CRUD**: `POST/GET/PUT/DELETE /api/accounts`
- **계산 잔액·검증**: `GET /api/accounts?include=calculatedBalance`  
  - 계정별 `inputBalance`(balance), `calculatedBalance`, `difference` 반환

### 2-3. UI/IA

- **자산(Accounts) 페이지** (`/accounts`)
  - 계좌 목록 (이름, 유형, 입력 잔액, 계산 잔액, 차이)
  - 총 자산(입력), 총 자산(계산)
  - 계정 추가·수정·삭제
  - 계정별 기준일·기준일 0시 잔액 설정
- **설정 > 자산**: 기존처럼 현재 잔액 입력 유지 (자산 페이지로 이동 링크 가능)
